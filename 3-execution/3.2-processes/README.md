# 3.2 — Processes

**What a process is:** a skill that runs *automatically* (runner/schedule/event) = skill + rulebook +
self-learning + a runner, plus a health-check and a ping-on-failure. **The skill-vs-process test:** a runner
starts it without a human → process; must it be called → skill.

- **Golden example:** [`example-inbox-router/`](example-inbox-router/SKILL.md) — a fully generic process
  described as a pattern (trigger → action → where output goes), concept-first, no real scripts or configs.
