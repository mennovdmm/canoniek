# Registry — built components

> The single table of contents: **what · where · status**. It points at the docs; it is not the docs.
> Resolve paths through this index — never hardcode a path elsewhere.

| Component | Kind | Where (canonical home) | Status | Watched? | Notes |
|---|---|---|---|---|---|
| `<name>` | kernel / skill / process / infra / technical | `<path or repo>` | active / planned / archived | yes / no / n.a. | <1 line> |

## Conventions
- **One row per built component.** If it's not here, it isn't "done."
- **Status:** `active` (live) · `planned` (designed, not built) · `archived` (retired, kept for history).
- **Watched?** `yes` for processes (must have a health-check) · `n.a.` for static components.
- A governance check compares this table against reality and flags drift or dangling references.
