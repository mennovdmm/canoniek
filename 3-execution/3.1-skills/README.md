# 3.1 — Skills

**What a skill is:** an on-demand, repeatable way of working (a capability) — a `SKILL.md` (with a sharp
auto-load `description`) + a `rulebook.md`. Status lives in the [registry](../../2-system/2.4-registry.md),
not in the folder. **The rulebook mechanic:** every correction → a dated rulebook line → next run loads it →
better.

- **Format:** the [skill template](../3.3-templates/SKILL-TEMPLATE.md) + the [rulebook template](../3.3-templates/RULEBOOK-TEMPLATE.md).
- **Golden example:** [`example-draft-reply/`](example-draft-reply/SKILL.md) — a fully generic skill that shows
  every part of the format. Copy the shape, not the content.
