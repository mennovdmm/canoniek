## What this doc must cover
- **Why continuous self-watching:** a system that extends itself needs gates so nothing silently rots —
  health, governance, self-healing, stuck-detection.
- **The hard routing rule:** an alert never goes *only* to the user — it goes to someone/something that can
  *act*, so problems get handled, not just signaled.
- **What gates typically enforce (as principles):** a skill/process without a rulebook → flag; a process
  without a health-check → flag; registry/doc drift or a dangling reference → flag; a secret in a commit →
  block.
- **One golden example** (anonymized): a single gate described end-to-end as a pattern — trigger, check,
  where the finding routes — *not* real code.
