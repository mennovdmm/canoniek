# 2.3 — Monitoring (the principle)

> **Tier 2 — principle, not implementation.** This doc ships the *why* and the *pattern* of self-watching, not
> scripts, schedulers, or addresses. A system that extends itself needs gates so nothing silently rots — but
> the concrete gate is something you (and Claude Code) build per setup. One anonymized example below shows the
> shape; copy the pattern, not code.

## Why continuous self-watching
The build protocol's "one rule" is only as good as its enforcement. Left to goodwill, rulebooks go unwritten,
the registry drifts, processes fail silently, and the system slowly becomes the monster it was meant to avoid.
**Gates make the discipline self-enforcing** — they catch what humans forget, so the system keeps its own
house in order: health, governance, self-healing, stuck/wedge detection.

## The hard routing rule
**An alert never goes *only* to the user.** A finding routes to **someone or something that can act on it** —
a maintainer agent, a queue, an inbox that gets worked — so problems get *handled*, not just signaled. A
notification to a human who can't fix it is fine *in addition*, never *instead*.

## What gates typically enforce (as principles)
- A skill/process without a `rulebook.md` → **flag** (learnings would evaporate).
- A process without a health-check → **flag** (silent failure).
- Registry/doc out of sync, or a dangling reference → **flag** (drift).
- A lifecycle phase outside the fixed vocabulary → **flag** (inconsistent state).
- A secret about to be committed → **block** (hard stop).

## One golden example (anonymized — the *pattern*, not code)
A "rulebook gate":
1. **Trigger** — runs on a schedule (say, daily) and on every change to a skill/process folder.
2. **Check** — for each skill/process, does a `rulebook.md` exist *and* is there a registry entry?
3. **On failure** — it doesn't email the user and stop. It drops a finding into the **maintainer's inbox**
   ("`<component>` has no rulebook — created a stub, please confirm") *and* nudges the maintainer agent to act.
   Optionally it self-heals (creates the stub) so the gap is closed, not just reported.
4. **Result** — the discipline holds without anyone remembering to check.

> That's the whole point of Tier 2: you learn *what* a gate must guarantee and *why*, then build the real one
> for your own stack. The model ships the contract, not the plumbing.
