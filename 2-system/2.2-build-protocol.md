## What this doc must cover
- **The one rule:** a change is *done* only when it is (1) classified, (2) in its canonical home, (3) in the
  registry, and (4) — if it's a process — watched. Otherwise it doesn't exist.
- **Classify → home:** the component taxonomy (Kernel → core · Skill → skills/ · Process → processes/ +
  runner · Knowledge → external store + search · State → DB / state files · Technical → repo + README ·
  Infra → infra/).
- **The skill-vs-process test:** does a runner/schedule/event start it without a human → process; must it be
  called → skill (even if it "automates"). No runner = no process.
- **Definition of done per type** + the always-do list (registry, log, no secrets, 1:1 before upgrade).
- **Self-learning:** a correction becomes a dated rulebook line, immediately — the chat is not memory.
- **Documentation discipline:** living sources, never static lists that rot; the registry is the one manual
  index.
