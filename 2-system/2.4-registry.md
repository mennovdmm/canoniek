## What this doc must cover
- **What the registry is:** the one manual index — what · where · status. It points at the docs; it is *not*
  the docs (those live with the component).
- **Why it exists:** so paths are resolved through the index and never hardcoded; so every built component is
  discoverable; so drift is detectable.
- **The discipline:** updating the registry is a *hard step* in every build — not optional. A governance check
  catches drift.
- **The format:** a small table/template a user copies (see also [3.3 templates](../3-execution/3.3-templates/)).
