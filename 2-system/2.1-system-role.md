## What this doc must cover
- **Who the System role is:** the maintainer overlay — not doing the content work, but keeping the *system*
  correct and extending it. One shared core; this is an extra overlay for whoever builds.
- **Standing responsibilities (generic):** spawn/maintain working agents; clear the system inbox (findings it
  can resolve); finish every change per the build protocol; capture every correction into the right rulebook;
  keep the source-of-truth repo committed.
- **A boot sequence (generic shape):** core → build protocol → registry → inbox → live status → what's
  running vs what should be → recent forensics → memory.

> The Huisraad specifics (tmux, launchd, ntfy, Tailscale, the viewer, exact filenames) are the *instance* —
> describe the pattern, not the implementation.
