# 2 — System

**The builder role.** How the system safely extends itself and keeps watching itself, without turning into a
monster of technical debt. One shared core (layer 1) plus a *role overlay* for whoever is building: classify
→ canonical home → registry → watch. This layer ships the **protocol and the principle**, never the concrete
scripts, hosts or configs.

- [2.1 — System role](2.1-system-role.md) — the builder overlay on top of the core.
- [2.2 — Build protocol](2.2-build-protocol.md) — the one rule + definition of done + the skill-vs-process test.
- [2.3 — Monitoring](2.3-monitoring.md) — gates/checks as a *principle + why* (Tier 2, no scripts).
- [2.4 — Registry](2.4-registry.md) — the registry discipline (the index, not the docs).
