# 1.3 — Identity placeholders

> **Stub — to be built (Tier 1).** Source: derived from the identity section of `core` (`l1.core` §1). These
> are the slots the bootstrap interview ([3.4](../3-execution/3.4-bootstrap/)) fills in to generate a user's
> first `core`. Each slot pairs a placeholder with the bootstrap question that fills it.

## What this doc must cover
A table of identity slots, each with: the placeholder token, the bootstrap question, and a one-line note on
how it's used. At minimum:
- **Who you are** — role, focus, expertise.
- **Tone / voice** — how you communicate, what to avoid.
- **Your business** — what it does, who its customers are, the customer-of-the-customer.
- **Your team / collaborators** — for routing (kept generic; no real names shipped).
- **Your data homes** — where knowledge, state and files live (external).
- **Your channels & tools** — how work comes in, which tools are wired (user-specific).

> Ship the *slots and the questions*, never a filled-in identity.
