## What this doc must cover
- **The three-part unit (the trinity):** every node — company, client, project, skill — is the same memory
  unit with three parts: **Charter** (identity/rules, semantic), **State** (what-is-where + dated history,
  working/episodic), **Rulebook** (learnings from corrections, procedural — the motor under the other two).
- **The layers as a tree:** a general layer (Core · Skills · Processes · Infra) shared across all instances,
  and a content layer (Client → Project → Subproject, one-to-many) that lives *outside* the repo.
- **The two movements:** cascade ↓ (context flows down a branch — a read-discipline, not native auto-load)
  and flywheel ↑ (corrections flow up into rulebooks — the self-learning loop).
- **The core principle:** *ship the schema, not the data* — structure is universal, content is personal.
- **The agent as cursor:** an incoming signal opens one branch; the agent reads up for *enough* context and
  writes up (progress + learnings), never loading the whole tree.
